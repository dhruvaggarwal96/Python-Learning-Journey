# Python Learning Journey - Day 8

## Topic: Exception Handling in Python

Today I learned how to make Python programs more robust and user-friendly by handling runtime errors using Exception Handling.

## Concepts Covered

### 1. Exceptions
- Understanding runtime errors
- Why programs crash
- Common exceptions in Python

### 2. try and except
- Handling errors without stopping the program
- Catching specific exceptions

### 3. else Block
- Executing code when no exception occurs

### 4. finally Block
- Running cleanup code regardless of errors

### 5. Common Exceptions
- ValueError
- ZeroDivisionError
- FileNotFoundError

## Key Learnings

- Programs should handle errors gracefully.
- Specific exceptions are better than generic exceptions.
- The finally block always executes.
- File operations should always be protected using exception handling.
- User input should never be trusted without validation.
